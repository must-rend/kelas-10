SLIDE 2 - LATAR BELAKANG & TUJUAN
========================================

Di era digital saat ini, memiliki sebuah portfolio online merupakan hal
yang sangat penting, terutama bagi seorang developer. Portfolio online
berfungsi sebagai "kartu nama digital" yang dapat menampilkan identitas,

Tujuan dari project ini adalah:
1. Membuat website portfolio yang profesional, modern, dan responsif.
2. Menerapkan arsitektur Fullstack dengan memisahkan antara frontend
   dan backend (decoupled architecture).
3. Mengimplementasikan sistem autentikasi untuk admin dashboard.
4. Menyediakan fitur Contact Form agar pengunjung dapat mengirim pesan
   langsung melalui website.


========================================
SLIDE 3 - TEKNOLOGI YANG DIGUNAKAN
========================================

Project ini dibangun menggunakan dua teknologi utama yang saling
terintegrasi:

BACKEND (Server-Side):
- Laravel 11 — Framework PHP yang powerful untuk membangun REST API.
- Laravel Sanctum — Sistem autentikasi berbasis token untuk API.
- SQLite — Database ringan yang digunakan untuk menyimpan data.
- Eloquent ORM — Untuk pengelolaan query database secara efisien.

FRONTEND (Client-Side):
- React.js 18 — Library JavaScript untuk membangun antarmuka pengguna.
- React Router DOM — Untuk navigasi antar halaman secara SPA
  (Single Page Application).
- Framer Motion — Library animasi untuk transisi dan efek visual.
- Lucide React — Icon library modern yang ringan.
- Vite — Build tool yang sangat cepat untuk development React.

ARSITEKTUR:
- Frontend dan Backend berjalan secara terpisah (decoupled).
- Komunikasi antara keduanya dilakukan melalui REST API.
- Frontend berjalan di port 5173, Backend di port 8000.

SLIDE 4 - FITUR-FITUR UTAMA
========================================

Website portfolio ini memiliki beberapa fitur utama, yaitu:

1. HALAMAN UTAMA (Landing Page)
   - Hero Section: Menampilkan nama, role sebagai Fullstack Web Developer,
     dan deskripsi singkat dengan animasi yang menarik.
   - Tombol "Lihat Proyek" dan "Download CV" untuk navigasi cepat.

2. TENTANG SAYA (About)
   - Menjelaskan profil dan pendekatan saya dalam pengembangan web.
   - Terdapat 3 kartu fitur: Frontend Development, Backend Development,
     dan Clean Code.

3. PROYEK UNGGULAN (Projects)
   - Menampilkan daftar proyek yang diambil secara dinamis dari API backend.
   - Setiap proyek menampilkan gambar, deskripsi, dan tech stack yang
     digunakan.
   - Memiliki fallback data jika API belum tersedia.

4. KEAHLIAN (Skills)
   - Menampilkan keahlian yang dikategorikan menjadi Frontend, Backend,
     dan Tools.
   - Dilengkapi dengan progress bar animasi yang menunjukkan tingkat
     penguasaan masing-masing skill.
   - Data skill diambil secara dinamis dari API.

5. FORMULIR KONTAK (Contact)
   - Pengunjung dapat mengirim pesan melalui form yang terdiri dari
     Nama, Email, dan Pesan.
   - Data pesan dikirim ke backend melalui API dan disimpan di database.
   - Terdapat notifikasi sukses/gagal setelah pengiriman.

6. SISTEM AUTENTIKASI (Login & Register)
   - Halaman Login dan Register dengan desain modern dan glassmorphism.
   - Autentikasi menggunakan Laravel Sanctum dengan token-based system.
   - Validasi form di sisi frontend dan backend.

7. ADMIN DASHBOARD (Protected Route)
   - Halaman khusus yang hanya bisa diakses setelah login (Protected Route).
   - Menampilkan profil admin yang sedang login.
   - Menampilkan daftar pesan masuk dari pengunjung secara realtime.
   - Fitur Logout untuk mengakhiri sesi.

SLIDE 6 - API ENDPOINTS
========================================

Berikut adalah daftar API endpoint yang tersedia di backend:

ENDPOINT PUBLIK (Tanpa Login):
- POST   /api/register     --> Registrasi akun baru
- POST   /api/login        --> Login dan mendapatkan token
- GET    /api/projects     --> Mengambil daftar proyek
- GET    /api/skills       --> Mengambil daftar keahlian
- POST   /api/contacts     --> Mengirim pesan kontak

ENDPOINT TERPROTEKSI (Harus Login / Bearer Token):
- GET    /api/me           --> Mendapatkan profil user yang login
- POST   /api/logout       --> Logout dan menghapus token
- GET    /api/contacts     --> Melihat semua pesan masuk (admin)

========================================
SLIDE 8 - KEAMANAN
========================================

Aspek keamanan yang diterapkan dalam project ini:

1. TOKEN-BASED AUTHENTICATION
   - Menggunakan Laravel Sanctum untuk menghasilkan Personal Access Token.
   - Setiap request ke endpoint terproteksi harus menyertakan token
     di header Authorization.

2. PASSWORD HASHING
   - Password pengguna di-hash menggunakan bcrypt sebelum disimpan
     di database, sehingga tidak ada password yang tersimpan dalam
     bentuk plain text.

3. PROTECTED ROUTES
   - Halaman Dashboard hanya bisa diakses oleh user yang sudah login.
   - Jika belum login, user akan otomatis diarahkan ke halaman login.

4. VALIDASI INPUT
   - Validasi dilakukan di sisi backend (Laravel Validator) maupun
     di sisi frontend (required fields, email format).

5. TOKEN REVOCATION
   - Saat login baru, token lama otomatis dihapus untuk mencegah
     penyalahgunaan sesi.

SLIDE 9 - TANTANGAN & SOLUSI
========================================

Selama pengembangan, ada beberapa tantangan yang saya hadapi:

1. INTEGRASI FRONTEND & BACKEND
   Tantangan: Menghubungkan React (port 5173) dengan Laravel (port 8000)
   menghadapi masalah CORS (Cross-Origin Resource Sharing).
   Solusi: Mengkonfigurasi CORS di Laravel agar mengizinkan request
   dari frontend.

2. STATE MANAGEMENT AUTENTIKASI
   Tantangan: Mengelola status login user secara global di seluruh
   komponen React.
   Solusi: Menggunakan React Context API (AuthContext) untuk menyimpan
   data user dan token secara terpusat.

3. DATA DINAMIS vs STATIC
   Tantangan: Memastikan website tetap berjalan meskipun API belum
   tersedia atau database kosong.
   Solusi: Menerapkan fallback data (dummy data) yang akan ditampilkan
   jika API gagal merespon.
